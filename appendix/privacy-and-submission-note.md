Privacy and Submission Boundary

Public author name: Akan Muhametgali

This public repository intentionally excludes student identification numbers, the second group member’s private details, the signed Group Charter, individual reflection forms, LMS screenshots and submission receipts.

The following information belongs only in the private university submission:

The full names and student IDs of both group members.

The workshop session and tutor details required by the cover page.

The completed Group Charter in the official university format.

A separate Reflection on Group Work document from each member.

Each member’s individual presentation file and recording.

The LMS submission confirmation.

The repository must not contain employee personal email addresses, private telephone numbers, account data, credentials, breach data, browser sessions, cookies, leaked information or claims based on attempts to access a system.

Academic boundary

The project is a cyberfraud risk assessment based on passive public information. It does not establish that FAB, an employee, a supplier or a customer has been compromised. It does not authorise security testing and must not be used as a plan for intrusion, impersonation or contact with the organisation.

Submission check

The private report should use Arial throughout, contain both group members’ required details and include the official Group Charter as an appendix. Each group member must separately submit a reflection and a 5–7 minute MP4 presentation with their face visible. The recording must be below the LMS file-size limit and should be replayed after export before upload.

Repository publication check

Before any public commit, confirm that the file does not contain a student ID, private group document, personal contact information, access token, credential, private screenshot or unverified accusation.
